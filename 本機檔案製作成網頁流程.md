# 本機檔案製作成網頁流程

這份流程適合把電腦裡的文字、圖片、PDF、簡報截圖、課程素材或其他檔案整理成一個可開啟、可分享、可部署的靜態網頁。

## 1. 建立專案資料夾

建議每個網站都用一個獨立資料夾管理：

```text
網站資料夾/
  index.html
  styles.css
  script.js
  assets/
    images/
    documents/
    downloads/
  site-dist/
```

目前這個資料夾已經有基本網頁檔案：

- `index.html`：網頁內容
- `styles.css`：版面與視覺樣式
- `script.js`：互動功能
- `assets/`：圖片與素材
- `.openai/hosting.json`：部署設定

## 2. 整理本機檔案

先把素材放進 `assets/`，並依用途分資料夾：

```text
assets/
  images/       放 JPG、PNG、WebP、GIF
  documents/    放 PDF、Word、簡報等可閱讀檔案
  downloads/    放希望讓訪客下載的檔案
```

檔名建議使用英文、數字和連字號，避免空白與特殊符號：

```text
good: course-intro.pdf
good: unit-01-cover.png
avoid: 第一單元 課程介紹(最新版).pdf
```

## 3. 決定網頁形式

依檔案類型選擇呈現方式：

| 本機檔案類型 | 建議做法 |
| --- | --- |
| 文字、筆記、課程說明 | 轉成 HTML 區塊 |
| 圖片 | 放入 `assets/images/`，用 `<img>` 顯示 |
| PDF | 放入 `assets/documents/`，用連結或嵌入預覽 |
| PowerPoint / Word | 先輸出成 PDF，或擷取重點做成網頁段落 |
| Excel / CSV | 整理成表格，必要時再加篩選或搜尋 |
| 下載檔 | 放入 `assets/downloads/`，用下載按鈕連結 |

## 4. 建立首頁內容

最簡單的 `index.html` 結構如下：

```html
<!doctype html>
<html lang="zh-Hant">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>網站名稱</title>
    <meta name="description" content="網站簡短說明" />
    <link rel="stylesheet" href="styles.css" />
  </head>
  <body>
    <header>
      <h1>網站名稱</h1>
      <p>這裡放網站或課程的簡短介紹。</p>
    </header>

    <main>
      <section>
        <h2>主要內容</h2>
        <p>把本機檔案整理成清楚的網頁內容。</p>
      </section>

      <section>
        <h2>檔案下載</h2>
        <a href="assets/downloads/example.pdf" download>下載檔案</a>
      </section>
    </main>

    <script src="script.js"></script>
  </body>
</html>
```

## 5. 加入圖片、PDF 和下載檔

圖片：

```html
<img src="assets/images/course-cover.png" alt="課程封面" />
```

PDF 連結：

```html
<a href="assets/documents/course-handout.pdf" target="_blank" rel="noopener">
  開啟講義 PDF
</a>
```

下載按鈕：

```html
<a href="assets/downloads/course-files.zip" download>
  下載課程附件
</a>
```

## 6. 本機預覽

如果網站只有 HTML、CSS、圖片和簡單 JavaScript，可以直接打開 `index.html` 預覽。

若要模擬正式網站環境，建議在資料夾內啟動本機預覽伺服器：

```powershell
python -m http.server 8000
```

然後在瀏覽器開啟：

```text
http://localhost:8000
```

## 7. 上線前檢查

發布前檢查這幾件事：

- 首頁可以正常開啟
- 圖片都顯示正確
- PDF 或下載連結都能開啟
- 手機寬度下文字不會擠在一起
- 檔名與路徑大小寫一致
- 沒有連到自己電腦的絕對路徑，例如 `C:\Users\...`
- 不需要公開的私人資料已移除

## 8. 打包發布檔

建立 `site-dist/`，放入要上線的檔案：

```text
site-dist/
  index.html
  styles.css
  script.js
  assets/
```

這個資料夾可以交給 Netlify、Vercel、Cloudflare Pages、GitHub Pages，或其他靜態網站服務使用。

這個專案已經附上一個打包工具，可以直接產生 `site-dist/`：

```powershell
powershell -ExecutionPolicy Bypass -File .\製作網頁發布檔.ps1
```

執行後會複製以下內容：

- `index.html`
- `styles.css`
- `script.js`
- `assets/`
- `本機檔案製作成網頁流程.md`

## 9. 建議工作節奏

每次製作網頁時，照這個順序做：

1. 收集本機檔案
2. 移到 `assets/` 並重新命名
3. 決定哪些內容要顯示、哪些只提供下載
4. 更新 `index.html`
5. 調整 `styles.css`
6. 需要互動時再更新 `script.js`
7. 本機預覽
8. 修正手機版與連結問題
9. 打包到 `site-dist/`
10. 發布網站

## 10. 可重複使用的檔案清單

每個網頁專案建議至少保留：

- `index.html`
- `styles.css`
- `script.js`
- `assets/`
- `本機檔案製作成網頁流程.md`

這樣下次要把新的本機檔案變成網頁時，只要複製這個資料夾，替換內容與素材即可。
